## Verification steps

The zip file distributed with this package is built directly from source via
GitHub Actions (https://github.com/RajwanYair/RegiLattice/actions/workflows/release.yml)
and published as a release asset on GitHub.

To verify:

1. Locate the release for this package version:
   https://github.com/RajwanYair/RegiLattice/releases/tag/v<version>

2. Download the `RegiLattice-<version>-win-x64.zip` asset from that release.

3. Compute its SHA-256 hash:
   Get-FileHash RegiLattice-<version>-win-x64.zip -Algorithm SHA256

4. Compare the hash against the value in `SHA256SUMS.txt` in the same release,
   and against the `checksum64` value in `chocolateyInstall.ps1` in this package.

All three values must match.

## Build reproducibility

The release workflow is defined in:
  https://github.com/RajwanYair/RegiLattice/blob/main/.github/workflows/release.yml

The `dotnet publish` step sets `-c Release -r win-x64 --self-contained true -p:PublishSingleFile=true`.
The version is determined solely by the git tag that triggered the workflow.
